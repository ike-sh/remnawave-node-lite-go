## v0.8.30 — 资源与运维优化

### 改进

- **`GOMEMLIMIT` 内化**：仅 `LOW_MEMORY=1` 时进程自动设 180MiB 软上限；大节点不再被 systemd 默认误限。
- **rw-core 日志轮转**：10MB 自动轮转，保留一份 `.1` 备份，防小盘打满。
- **网卡速率轮询**：采样间隔 1s → 3s，降低空闲唤醒。
- **配置 JSON 缓存**：unix socket get-config 复用缓存，减少大配置反复序列化。
- **`xray version` 探测**：版本已知后 health 不再每次 fork。

### 修复

- 网卡 rx/tx 计数器回绕时跳过异常采样，避免离谱速率。

### 升级

```bash
curl -fsSL https://raw.githubusercontent.com/ike-sh/remnawave-node-lite-go/v0.8.30/scripts/upgrade.sh | sudo bash -s -- --yes
sudo systemctl restart remnawave-node
```

> 升级后 systemd 单元不再默认注入 `GOMEMLIMIT`；低内存节点请确保 `node.env` 中 `LOW_MEMORY=1` 已设置（`install-node.sh --low-memory` 会自动写入）。

完整变更见 [CHANGELOG.md](../CHANGELOG.md).
