## 重要：修复服务器重启后无法自动上线

v0.8.24 已将 xray 配置持久化到 `/var/lib/remnanode/last-start.json`，但进程收到 SIGTERM（关机/重启）时 `Stop()` 会误删该文件，导致下次启动 `RestoreOnBoot` 找不到配置，Panel 显示离线，需手动禁用/启用节点。

**v0.8.25 修复此问题：正常关机/重启保留 `last-start.json`，仅 Panel 禁用节点（`/node/xray/stop`）时清除。**

### 修复

- **SIGTERM 误删持久化配置**：`Stop(clearPersist)` — 进程关机 `Stop(false)` 保留配置；Panel 禁用 `Stop(true)` 清除。
- 与 v0.8.24 的 `RestoreOnBoot` 配合，服务器重启后 rw-core 可自动恢复。

### 升级

```bash
# Alpine
curl -fsSL https://raw.githubusercontent.com/ike-sh/remnawave-node-lite-go/v0.8.25/scripts/install-node-alpine.sh -o /tmp/install-alpine.sh
bash /tmp/install-alpine.sh --upgrade --yes
rc-service remnawave-node restart

# systemd
curl -fsSL https://raw.githubusercontent.com/ike-sh/remnawave-node-lite-go/v0.8.25/scripts/upgrade.sh | sudo bash -s -- --yes
sudo systemctl restart remnawave-node
```

**验证**：节点在线且 Panel 至少成功 `xray/start` 一次（生成 `last-start.json`）后，重启服务器，Panel 应自动显示在线，无需手动禁用/启用。
